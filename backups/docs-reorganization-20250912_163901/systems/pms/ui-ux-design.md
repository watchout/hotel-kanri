# hotel-pms UI/UX設計

## 概要
hotel-pmsのUI/UX設計は、フロント業務の効率化と正確性を最優先に考慮しています。24時間稼働するホテル運営の中核システムとして、直感的で迅速な操作が可能なインターフェースを提供します。

## デザイン原則

### 1. 効率性
- **最小クリック数**: 主要タスクは3クリック以内で完了
- **ショートカットキー**: 頻繁に使用する機能にキーボードショートカット
- **バッチ処理**: 一括チェックイン/アウトなどの効率化機能

### 2. 明確性
- **視覚的階層**: 重要情報を目立たせる設計
- **カラーコーディング**: ステータスや優先度の視覚的表現
- **一貫性**: 共通のデザインパターンと用語

### 3. エラー防止
- **リアルタイム検証**: 入力時のバリデーション
- **確認ダイアログ**: 重要操作の確認
- **ダブルブッキング防止**: 視覚的警告と自動検知

### 4. 柔軟性
- **レスポンシブデザイン**: 様々な画面サイズに対応
- **カスタマイズ可能**: ユーザー設定によるUI調整
- **拡張性**: 新機能の追加に対応

## デザインシステム

### カラーパレット
- **プライマリ**: #2C3E50（ダークブルー）
- **セカンダリ**: #3498DB（ライトブルー）
- **アクセント**: #E74C3C（レッド）
- **成功**: #2ECC71（グリーン）
- **警告**: #F39C12（オレンジ）
- **エラー**: #E74C3C（レッド）
- **背景**: #F5F7FA（ライトグレー）
- **テキスト**: #333333（ダークグレー）

### タイポグラフィ
- **見出し**: Noto Sans JP, Bold
- **本文**: Noto Sans JP, Regular
- **アクセント**: Noto Sans JP, Medium
- **サイズ階層**: 12px, 14px, 16px, 18px, 24px, 32px

### コンポーネント
- **ボタン**: 主要アクション、二次アクション、テキストリンク
- **フォーム**: 入力フィールド、選択メニュー、チェックボックス
- **カード**: 情報表示、アクション付きカード
- **テーブル**: データ表示、ソート機能付き
- **モーダル**: 確認、詳細表示、フォーム入力
- **通知**: 成功、警告、エラー、情報

## ユーザーフロー

### チェックインフロー
1. 予約検索（予約番号/顧客名）
2. 予約詳細確認
3. 顧客情報確認・更新
4. 部屋割り当て
5. 支払い確認・追加料金
6. キーカード発行
7. チェックイン完了

### チェックアウトフロー
1. 部屋番号/顧客名検索
2. 追加料金確認
3. 最終請求書作成
4. 支払い処理
5. 領収書発行
6. チェックアウト完了
7. 清掃ステータス更新

### 予約作成フロー
1. 空室確認
2. 顧客情報入力
3. 部屋タイプ・料金選択
4. 追加サービス選択
5. 予約内容確認
6. 予約完了・確認メール送信
