# AI 中断プロトコル（常時適用ルール）

> ai-dev-framework v3.0 準拠
> 詳細: docs/standards/AI_ESCALATION_PROTOCOL.md

---

## 7つの中断トリガー

以下のいずれかに該当したら、AIは即座に作業を中断しユーザーに質問する。

| トリガー | 条件 | 対応 |
|:---------|:-----|:-----|
| **T1** | SSOT に記載がない仕様判断 | 層による |
| **T2** | SSOT の記載が曖昧 | 層による |
| **T3** | 技術的な選択肢が複数ある | 条件付き進行 |
| **T4** | SSOTと既存実装の矛盾 | **常に停止** |
| **T5** | 制約・規約に未定義のケース | 条件付き進行 |
| **T6** | 影響範囲が不明 | **常に停止** |
| **T7** | ビジネス判断が必要 | 条件付き進行 |

---

## SSOT 3層構造による判断

```
不明点を発見
  │
  ├── CORE層 or CONTRACT層か？ → Yes → 停止して質問
  │
  ├── T4 or T6 か？ → Yes → 常に停止
  │
  ├── DETAIL層か？ → Yes → デフォルト案で進む
  │   │                     + Decision Backlog に記録
  │   │
  │   └── 影響が大きいか？ → Yes → Feature Flag で隠す
  │                         → No → そのまま実装
  │
  └── 判断できない → 停止して質問
```

### 層の分類

| 層 | 含まれるもの | 変更ルール |
|:---|:-------------|:-----------|
| **CORE** | 目的、スコープ、ユーザーストーリー、主要フロー、ビジネスルール | 変更時は承認必須 |
| **CONTRACT** | API仕様、画面一覧・遷移、DB主要テーブル、状態遷移図 | 破壊的変更禁止 |
| **DETAIL** | バリデーション詳細、UI文言、エラーメッセージ、非機能要件微調整 | 自由に変更可 |

## 中断時の報告フォーマット

```markdown
⚠️ 確認が必要です

トリガー: [T1-T7]
発生箇所: [何をしている時か]

質問: [具体的な質問]

選択肢:
a) [選択肢A] → [影響]
b) [選択肢B] → [影響]

推奨: [あれば]
```

---

## 進行時の記録

DETAIL層で進む場合:

1. `docs/ssot/DECISION_BACKLOG.md` に記録
2. コードに `// DECISION: DB-XXX` コメント
3. セッション終了時にユーザーへ報告

---

## 禁止行動

```
❌ 「おそらく○○だろう」と推測して実装する
❌ CORE/CONTRACT層の不明点をデフォルトで進める
❌ T4（矛盾）, T6（影響不明）を無視して進める
❌ Decision Backlog に記録せずにデフォルト実装する
```
