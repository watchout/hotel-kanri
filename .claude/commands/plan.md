# /plan コマンド

タスクの実装計画を立案します。

## 使い方

```
/plan [タスクID or 機能名]
```

例:
```
/plan DEV-0170
/plan 有人ハンドオフ機能
```

## 実行フロー

1. **スコープ確認**
   - タスク目的の明確化
   - 関連SSOT特定
   - 影響範囲把握

2. **実装状況スキャン**
   - Git管理下ファイル確認
   - コミット履歴確認
   - 未コミット変更確認

3. **タスク分解**
   - Item/Step形式で構造化
   - 依存関係明示
   - 各Itemに工数見積もり

4. **リスク洗い出し**
   - 技術的リスク
   - スケジュールリスク

## 参照ドキュメント

- agents/planner.md

## 出力形式

```markdown
## 実装計画: [タスク名]

### 参照SSOT
- SSOT_XXX.md

### 依存関係
- [前提タスク]

### 実装アイテム

#### Item 1: [名前]（見積もり: X時間）
- Step 1: ...
- Step 2: ...

### リスク
- [リスク]

### 合計工数
X時間
```
